WCF

De Windows Communication Foundation (of WCF) is een verzameling van .NET-technologieën voor het bouwen en het gebruiken van netwerksystemen.

Via deze technologieën is het mogelijk om een webservice te bouwen en te hosten. Hierna kunnen andere applicaties met deze web service communiceren. Visual studio kan deze webservice voor je hosten en biedt je een WCF Test Client applicatie om te testen of je webservice goed werkt.

Het template project in Visual Studio voor een WCF applicatie bevat standaard de volgende onderdelen.

  • Een service interface
  • Een service implementatie van deze interface
  • Een datacontract waar de service gebruik van maakt

Service Interface

Om een service beschikbaar te maken voor andere applicaties hebben we een service interface nodig

Dit is een interface met bijbehorende ServiceContract attribuut. Met dit attribuut geef je aan dat deze service beschikbaar is voor client applicaties om te consumeren. Hoe je een service moet consumeren komen we later op terug. 

[ServiceContract]
public interface IChatService
{

    [OperationContract]
    Message[] SendMessage(Message message);

    [OperationContract]
    Message[] GetMessages();
}

Een service heeft een aantal methodes, zogenoemde contracts. Dit zijn methodes waar we later gebruik van kunnen maken. Deze methodes moet je labelen met een Operation Contract attribuut.

De implementatie van deze interface is een .svc file. Als je de file opent ziet deze er uit als een normale klasse. WCF zal deze implementatie gebruiken in de service die hij gaat hosten.

Service Data

Naast de methodes die we kunnen gebruiken op een service, komen er vaak ook complexe data objeecten bij kijken. Ook deze objecten kun je toegankelijk maken voor client applicaties. De klasse die je toegankelijk wilt maken voor deze applicaties worden gelabeld met een Data Contract attribuut.

Daarna kun je kiezen welke properties je van dit object beschikbaar wilt maken. Dit doe je met het bijbehorende Data Member attribuut. 

[DataContract]
public class Message
{

    [DataMember]
    public string Content { get; set; }

    [DataMember]
    public DateTime TimeStamp { get; set; }

    [DataMember]
    public string User { get; set; }
}

Als alle Service, Operation en Data contracten geformuleerd zijn, kunnen we onze applicatie gaan testen. Gebruik hiervoor de WCF test client.

Deze client start niet altijd op als je de service runt. Dit gebeurd alleen als je de servicenaam.svc file selecteert voordat je de applicatie start. Daarnaast zijn er andere manieren om de WCF test client te starten.
