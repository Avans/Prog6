Bouw een Rock-Paper-Scissors-Lizard-Spock (RPSLS) game service. Dit is een spel variant op rock paper scissors. Voor deze opdracht heb je geen database nodig.

De gebruiker kan via een WPF applicatie een symbool naar de server sturen. Naast het symbool stuurt hij ook zijn naam, zodat de server weet wie het symbool gestuurd heeft. De server genereert nu zelf een nieuw symbool, en controleert wie er gewonnen heeft. Als resultaat stuurt de server beide symbolen en bijbehorende spelers (jij en computer).

Je applicatie bestaat uit de volgende onderdelen:

WCF service

Een WCF service waar de gebruiker een potje RPSLS kan spelen tegen de computer.

  • Deze service heeft een bijbehorende Enum Symbool.
  • Deze service heeft een bijbehorend datacontract als resultaat met daarin minstens:
    • Symbool van de gebruiker
    • Symbool van de computer
    • Tijdstip van het spel
    • De winnaar

Schrijf unit tests om te controleren of de business logica voor het vergelijken van symbolen goed werkt.

Frontend applicate

Een WPF applicatie met de bijbehorende knoppen en een manier om een gebruikersnaam te kiezen.

Deze applicatie consumeert de eerder gemaakte webservice.

Extra: Multiplayer

Voeg een extra OperationContract toe aan je web service waarmee de gebruiker een multiplayer spel kan spelen.

Als een gebruiker een symbool verstuurt moet je kijken wat het vorige geposte symbool is. Is deze van dezelfde gebruiker? Dan gebeurt er niks. Is deze van een andere gebruiker? Dan kun je controleren of deze gewonnen heeft of niet. We moeten dus elke keer een verstuurd symbool kunnen bewaren.
