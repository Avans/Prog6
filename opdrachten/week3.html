Bouw een Rock-Paper-Scissors-Lizard-Spock (RPSLS) game service. Dit is een spel variant op rock paper scissors. Voor deze opdracht heb je geen database nodig.

De gebruiker kan via een WPF applicatie een symbool naar de server sturen. Naast het symbool stuurt hij ook zijn naam, zodat de server weet wie het symbool gestuurd heeft. De server genereert nu zelf een nieuw symbool, en controleert wie er gewonnen heeft. Als resultaat stuurt de server beide symbolen en bijbehorende spelers (jij en computer).

Je applicatie bestaat uit de volgende onderdelen:

WCF service

Een WCF service waar de gebruiker een potje RPSLS kan spelen tegen de computer.

  • Deze service heeft een bijbehorende Enum Symbool.
  • Deze service heeft een bijbehorend datacontract als resultaat met daarin minstens:
    • Symbool van de gebruiker
    • Symbool van de computer
    • Tijdstip van het spel
    • De winnaar

Schrijf unit tests om te controleren of de business logica voor het vergelijken van symbolen goed werkt.

Frontend applicate

Een WPF applicatie met de bijbehorende knoppen en een manier om een gebruikersnaam te kiezen.

Deze applicatie consumeert de eerder gemaakte webservice.

Extra: Multiplayer

Voeg een extra OperationContract toe aan je web service waarmee de gebruiker een multiplayer spel kan spelen.

Als een gebruiker een symbool verstuurt moet je kijken wat het vorige geposte symbool is. Is deze van dezelfde gebruiker? Dan gebeurt er niks. Is deze van een andere gebruiker? Dan kun je controleren of deze gewonnen heeft of niet. We moeten dus elke keer een verstuurd symbool kunnen bewaren.

We gaan een League manager bouwen. Dit is een spel waarbij je je eigen League of legends team kan samenstellen en kan spelen tegen andere spelers.

Om te beginnen moet je het Start project forken. In deze versie kun je met 2 man aan deze opdracht werken.

In de solution LeagueMananger zitten de volgende projecten

  • LeagueManager.Domain
  • LeagueManager.Service
  • LeagueManager.Test
  • LeagueManager.WPF

De flow van het spel

  1. Stel een team samen van champions
  2. Stuur deze samenstelling naar de server
  3. Kijk elke seconde of er een andere speler een samenstelling heeft verstuurd
  4. De server kijkt welke samenstelling gewonnen heeft en stopt deze in een game resultaat
  5. Haal de resultaten op van de server

De opdracht

Bouw de service verder af en sluit deze aan op de client-applicatie. Deze service moet een OperationContract hebben waarmee je je gekozen opstelling naar de server kan versturen. Daarnaast moet de server een OperationContract hebben waarmee je het resultaat van de 'match' kan ophalen. Indien de 'match' resultaten direct na het versturen van je opstelling beschikbaar zijn, moet deze methode de resultaten meteen teruggeven.

De service moet ook een OperationContract hebben die een lijst van alle te spelen champions teruggeeft.

In het domeinmodel zit een interface IChampion. Gebruik deze interface om voor minstens 5 champions een klasse te definiëren. Gebruik hierna Ninject om een volledige lijst met champions op te halen.

Technische eisen

  • Maak gebruik van Ninject of een ander Dependency Injection framework
  • Zorg voor minstens 60% code coverage
  • Host de service in WCF
  • Sla de informatie op in een database (ef)
