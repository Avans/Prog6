Rock Injection

Workshop assignment ws Workshop Assignment

Open de Visual Studio Solution DependencyInjectionStarter. Deze solution bestaat uit de volgende projects:

  • DependencyInjectionStarter (Console applicatie)
  • DependencyInjectionStarter.Library (Class Library)

Bekijk de class RockBand in het DependencyInjectionStarter.Library project. Deze class bevat een aantal muziekinstrumenten. Als je de method DoSoundCheck() van deze class aanroept wordt aan elk instrument gevraagd een geluid te maken. NB. Voor het gemak rekenen we een vocal ook als een muziekinstrument ;)

In de console applicatie wordt een instantie van de RockBand class aangemaakt en wordt de DoSoundCheck() method aangeroepen. De gemaakte geluiden verschijnen dan als tekst op de console. Zorg dat je de code snapt en run de applicatie om het resultaat te bekijken.

De opdracht

Het nadeel van de RockBand class is is dat deze tightly coupled is met de muziekinstrumenten. De opdracht is de Visual Solution te refactoren zodat de RockBand class een variabel aantal muziekinstrumenten kan bevatten. Maak hierbij gebruik van Dependency Injection via de constructor. Maak vervolgens om te bewijzen dan het werkt twee instanties van de RockBand class aan:

  1. Metallica deze band heeft de volgende instrumenten: (2x Gitaar, Basgitaar, Drums en Vocals)
  2. Coldplay deze band heeft de volgende instrumenten: (Gitaar, Basgitaar, Drums, Vocals en Keyboard)

Voer vervolgens de DoSoundCheck() method uit op beide instanties en controleer dat de juiste instrumenten afgespeeld worden.

In deze opdracht maak je een WPF applicatie maken voor blog posts, belangrijk is dat je gebruik maakt van het Repository Design Pattern met een Repository interface en Dependency Injection.

Het is verplicht om in deze opdracht gebruik te maken van het dependency injection framework http://www.ninject.org/. Daarnaast mag je GEEN gebruik maken van het MvvM Light framework.

Open het start project CrudRepo. In deze solution vind je 2 projecten.

  1. Een WPF applicatie
  2. Een Class Library met hierin de POCO Post

Het is aan jou om deze applicatie verder af te bouwen. Voeg de onderstaande onderdelen toe aan de applicatie.

  • Definieer een IPostRepository interface met de methods: GetAll, Add en Delete
  • Implementeer een DummyPostRepository, vul deze met minimaal 5 hardcoded blog posts
  • De klasse MainViewModel krijgt toegang tot de IPostRepository via de constructor.

NinjectServiceLocator

Nu is het aan jou de taak om ervoor te zorgen dat Ninject de juiste implementatie van het IPostRepository interface aan de viewmodels meegeeft. De Binding van de DataContext verwijst al naar de juiste klasse. In de klasse NinjectServiceLocator zul de juiste Ninject logica moeten gaan implementeren. Lees goed door de documentatie van Ninject en kijk naar voorbeelden op het internet.

Is het gelukt om de repository te injecten in het viewmodel?
